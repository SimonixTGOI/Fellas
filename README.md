# Fellas

Fellas is a Paper plugin that adds configurable kits, economy-based kit purchases, cooldowns, private messaging, and customizable MiniMessage responses.

## Features

* Configurable kits
* Free and paid kits
* Vault economy support
* Persistent kit cooldowns through `cooldowns.yml`
* Configurable messages
* MiniMessage color support
* Placeholder support for dynamic messages
* Kit permissions
* Kit list command
* Private message command
* Tab completion for supported commands

## Requirements

* Paper server
* Java compatible with your server version
* Vault, required only for paid kits
* An economy plugin, required only for paid kits

If Vault or an economy provider is not installed, free kits will still work. Paid kits will be disabled until economy support is available.

## Commands

| Command                    | Description                               |
| -------------------------- | ----------------------------------------- |
| `/kit <kit>`               | Claims a kit                              |
| `/kits`                    | Shows available kits                      |
| `/kitreload`               | Reloads kits and messages from the config |
| `/send <player> <message>` | Sends a private message to another player |

## Permissions

| Permission                  | Description                    |
| --------------------------- | ------------------------------ |
| `fellas.kit.<kit>`          | Allows claiming a specific kit |
| `fellas.kit.allkits`        | Allows access to all kits      |
| `fellas.kit.bypasscooldown` | Bypasses kit cooldowns         |
| `fellas.kits`               | Allows using `/kits`           |
| `fellas.kit.reload`         | Allows using `/kitreload`      |
| `fellas.send`               | Allows using `/send`           |

If a kit does not define a custom permission in the config, Fellas automatically uses:

```text
fellas.kit.<kit>
```

## Configuration

Example `config.yml`:

```yaml
kits:
  Starter:
    price: 0
    permission: fellas.kit.starter
    cooldown: 60
    items:
      - STONE_SWORD:1
      - BREAD:16

  PvP:
    price: 100
    permission: fellas.kit.pvp
    cooldown: 300
    items:
      - IRON_SWORD:1
      - COOKED_BEEF:16

messages:
  onlyPlayer: "<red>Only players may use this command.</red>"
  noPermission: "<red>Insufficient permissions.</red> <dark_purple>({permission})</dark_purple>"
  kitUsage: "<red>Usage: /kit [kit]</red>"
  kitNotFound: "<red>This kit does not exist.</red>"
  kitClaimed: "<dark_purple>Kit {kitName} has been claimed!</dark_purple>"
  kitPurchase: "<dark_purple>Kit {kitName} has been purchased!</dark_purple> <yellow>New balance:</yellow> <green>€{balance}</green>"
  kitCooldown: "<red>Kit still in cooldown, remaining time:</red> <dark_purple>{remainingCooldown}s</dark_purple><red>.</red>"
  notEnoughMoney: "<red>You don't have enough money to claim this kit.</red>"
  transactionFailed: "<red>Transaction failed.</red>"
  economyDisabled: "<red>Economy is temporarily disabled. This paid kit cannot be claimed.</red>"
  kitsHeader: "<dark_purple>Available kits:</dark_purple>"
  kitsNotOne: "<red>No kits available.</red>"
  kitReload: "<green>Kits reloaded.</green>"
```

## Cooldowns

Kit cooldowns are saved automatically in:

```text
cooldowns.yml
```

This file is generated by the plugin and keeps cooldowns active after server restarts.

## Installation

1. Download the plugin `.jar` from the latest release.
2. Place it inside your server's `plugins` folder.
3. Install Vault and an economy plugin if you want to use paid kits.
4. Start or restart the server.
5. Edit `config.yml` as needed.
6. Use `/kitreload` to reload the configuration.

## Building from Source

Clone the repository and build with Maven:

```bash
mvn clean package
```

The compiled plugin will be generated inside the `target` folder.
